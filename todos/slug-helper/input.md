# slug-helper — Input

Extract shared slug helper module (teleclaude/slug.py). Currently slug handling is scattered: SLUG_PATTERN lives in todo_scaffold.py, normalization is copy-pasted inline in \_handle_bugs_report and \_handle_content_dump, \_derive_slug lives in content_scaffold.py, and uniqueness is handled differently everywhere (FileExistsError in todos, counter suffix in content, bool return in roadmap). New module owns: validate_slug() for format check, normalize_slug() for text-to-slug conversion, ensure_unique_slug(base_dir, slug) for counter-suffix collision resolution. Callers to wire: create_todo_skeleton(), create_bug_skeleton(), remove_todo() in todo_scaffold.py; \_derive_slug() and create_content_inbox_entry() in content_scaffold.py; \_handle_bugs_report() and \_handle_content_dump() in telec.py; CreateTodoModal in modals.py. All import SLUG_PATTERN and validation from the new module. Todos get counter-suffix uniqueness like content already has.
