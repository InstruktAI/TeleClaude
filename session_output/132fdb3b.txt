> What are the main components of the TeleClaude architecture? 

● Based on the architecture documentation, here are the main components of
  TeleClaude:

  Core Architectural Layers

  1. Core Layer (teleclaude/core/)
  - adapter_client.py - Central hub for ALL adapter operations
  - db.py - Session persistence (module-level singleton)
  - terminal_bridge.py - tmux interaction
  - models.py - Data classes (Session, Recording)
  - command_handlers.py - Command routing logic
  - message_handler.py - User message handling
  - terminal_executor.py - Terminal command execution
  - events.py - Event type definitions
  - protocols.py - Protocol definitions (RemoteExecutionProtocol)

  2. Adapter Layer (teleclaude/adapters/)
  - base_adapter.py - Abstract interface with has_ui flag
  - telegram_adapter.py - Telegram Bot API (has_ui=True, UI platform)
  - redis_adapter.py - Redis Streams (has_ui=False, transport +
  RemoteExecutionProtocol)

  3. MCP Layer (teleclaude/mcp_server.py)
  - AI-to-AI communication via transport adapters
  - Uses AdapterClient for cross-computer messaging

  4. Main Daemon (teleclaude/daemon.py)
  - Coordination and lifecycle management
  - PID file locking
  - Initializes AdapterClient
  - Subscribes to events
  - Manages terminal output polling

  Key Architecture Patterns

  1. Observer Pattern - AdapterClient has NO daemon reference; daemon
  subscribes to events
  2. Module-Level Singleton - SessionManager (db) accessed via import
  3. Origin/Observer Broadcasting - One interactive adapter (origin), others
   read-only (observers)
  4. Protocol-Based Capabilities - RemoteExecutionProtocol for
  cross-computer orchestration
  5. AdapterClient as Central Hub - ALL adapter operations flow through it

  The system enables remote terminal access via Telegram/Redis with
  persistent sessions, real-time output streaming, and multi-adapter
  broadcasting.
