● Project Overview: TeleClaude

  TeleClaude is a multi-platform terminal bridge daemon that enables remote
  terminal access and AI-to-AI cross-computer communication.

  What it does:

  1. Remote Terminal Access via Telegram - Control your computer's terminal
  through Telegram topics/threads, making it possible to run commands, view
  output, and manage terminal sessions from your phone or any Telegram
  client.
  2. AI-to-AI Cross-Computer Communication - Uses Redis Streams to enable
  Claude Code instances on different computers to communicate and execute
  commands remotely on each other's machines.
  3. Persistent Terminal Sessions - Built on tmux, sessions survive daemon
  restarts and can be resumed at any time.
  4. Multi-Adapter Architecture - Extensible design supporting multiple
  platforms:
    - TelegramAdapter - Interactive UI via Telegram Bot API
    - RedisAdapter - Transport layer for AI-to-AI communication
    - Future: WhatsApp, Slack, REST API adapters
  5. Real-time Output Streaming - Polls terminal output every second and
  broadcasts to connected adapters with smart message editing (first 10s)
  and history preservation (after 10s).

  Key Architecture:

  - Python 3.11+ async daemon (asyncio-based)
  - Observer pattern - Daemon subscribes to adapter events
  - Protocol-based capabilities - Transport adapters implement
  RemoteExecutionProtocol
  - Module-level singleton - Database accessed via from teleclaude.core.db 
  import db
  - Origin/Observer broadcasting - One interactive adapter, others are
  read-only observers
  - Session persistence - SQLite database (teleclaude.db) stores sessions
  across restarts

  Current Status:

  The daemon is currently running (PID in teleclaude.pid), with active
  session output files in session_output/.
